The moment.

After they drop the file is where conversion happens. Five states. One flow.

— Purpose

Most demos throw away the work the visitor put in. Zili's demo flow keeps it — and uses the wow moment to ask for the signup. The states below show the full post-drop experience: loading, success, save prompt, dismissed, and error. Each is designed to feel like one continuous flow, not a sequence of screens.

— State 01

Loading. 2 to 4 seconds.

First, manage perceived speed. The visitor's confidence builds when they can see what's happening. Show step-by-step progress, not just a spinner. Generic spinners read as "broken or slow"; itemised progress reads as "working hard for me."

Why this works

Itemised progress > spinner

A spinner means nothing. A list of completed steps tells the user the system is working *for them*, not just *at them*. Every ✓ is a small win.

Brand discipline

The cursor is the only animation

One amber cursor blink, one amber progress bar. No spinners, no bouncing dots, no skeleton screens. Restraint reads as confidence.

Copy strategy

"Reading", not "Loading"

"Loading" is generic SaaS. "Reading your artifact" personalises the moment — Zili is paying attention to what you uploaded.

Failure mode handled

If a step fails, this is where it surfaces

The active step turns red instead of green. The progress bar stops. Error state (State 05) takes over. No mystery silence.

— State 02

Rendered. The wow moment.

The artifact appears, pixel-perfect, with the Zili presenter chrome already wrapped around it. A brief success banner at the top confirms what just happened. The visitor can navigate, fullscreen, and interact — exactly as if they were already a customer.

Why this works

Show the time, name the proof

"Rendered in 2.3 seconds" turns an abstract claim into specific evidence. The exact duration is more persuasive than "fast." Adding the slide count makes the detection feel intelligent.

Interaction works

This isn't a screenshot — it's the product

The presenter chrome is live. Arrow keys advance slides. F enters fullscreen. The visitor experiences the actual product, not a recorded demo. Their muscle memory begins forming.

Banner copy

Past tense, not future tense

"Rendered" is done. "Now rendering" or "Render complete" hedges. Editorial confidence: name the accomplishment plainly.

No CTA yet

Let them play, then ask

The save prompt does NOT appear immediately. The visitor gets ~8 seconds to navigate, click around, fullscreen. The wow needs space before the ask.

— State 03

The save prompt. ~8 seconds after render.

The most important moment of the entire marketing site. The artifact dims to 35% opacity (still visible — it's *their* work, not Zili's). A focused modal appears with a specific ask: keep this, get a shareable link, in two clicks. Google OAuth is primary; email is the alternative.

The artifact stays visible

Dimming the background to 35% (not 0% black overlay) keeps their work in view. The visitor sees what they're being asked to save. Removing it entirely would feel transactional.

Specific over generic

Two paths, one is faster

Honest dismiss

— State 04

Dismissed. Soft persistent reminder.

If they dismiss, the modal disappears and the artifact returns to full opacity. A small floating CTA stays in the corner — present but not nagging. They can keep exploring. The CTA only re-prompts the full modal if they try to leave the page or refresh.

Why this works

Soft presence, not nagging

The dismissed user keeps full agency — artifact at full opacity, all controls working. The floating CTA is small, in the corner, dismissable. It earns trust by not being aggressive.

Re-prompt trigger

On page-leave intent, the modal returns

If the visitor moves to close the tab or hit refresh, the full save modal returns ("Wait — you'll lose this"). One re-prompt only. Annoying past that.

Behind the scenes

The artifact is stored for 24h

Even if they walk away, the upload is retained for 24 hours under a temporary token. If they come back via the same browser, the same artifact is still there. Implementation note for engineering.

Floating CTA position

Bottom-right, never blocking content

Out of the visual centre. Easy to dismiss. Auto-hides if the user enters fullscreen. Re-appears on exit.

— State 05

Couldn't render. Honest, useful, recoverable.

When a render fails — file too large, malformed HTML, security violation, server error — the response is straightforward. Name the problem, suggest the fix, offer a retry path. Never let the visitor blame themselves for what's actually a Zili limitation.

That file is over 10MB.

Zili currently accepts HTML files up to 10MB. Most LLM artifacts are well under this — yours is probably bundled with images or a large CSS file. Try uploading the HTML alone, or split out the assets.

Why this works

Specific, not generic

"That file is over 10MB" tells them exactly what went wrong. "Something went wrong" is useless and erodes trust. Every error message is rewritten to be specific.

Suggest the fix

Don't just name the problem

Every error message includes the most likely fix ("try uploading the HTML alone"). The user shouldn't have to figure out how to recover.

Recovery is one click

"Try another file" is the primary CTA

The error state doesn't dead-end. The drop zone is one click away. No re-loading the page, no losing context.

Tone

Honest, not apologetic

No "Oops!" No emoji. No "We're so sorry!" The error is what it is. Editorial restraint applies even in failure states.

— For engineering

Implementation notes.

Specifics that Claude Code will need when building this flow. These are the small decisions that, if guessed wrong, would require redoing the work.

Timing

Save prompt delay: 8 seconds

Tunable in feature flags. Initial value 8s. If user interacts with the presenter chrome (clicks next/prev), reset the timer. Don't interrupt active exploration.

Temporary storage

Demo artifacts retain 24h, by browser cookie

No DB user record. Cookie holds a temp token mapping to R2 storage path. Cookie expires 24h. Object purges 25h. If user signs up within 24h, transfer ownership.

Re-prompt trigger

One re-prompt on beforeunload

If user dismissed save modal then attempts to leave page, beforeunload listener triggers the full modal once. Subsequent leave attempts go through without intercept.

Persistent CTA visibility

Hides in fullscreen, reappears on exit

Fullscreen API event listener toggles floating CTA visibility. Restore on fullscreenchange exit.

Error messages

All error copy in a dedicated dictionary

errors.ts file. Every error has a code, a user-facing title, a user-facing body, a suggested fix CTA. Centralised so copy iteration doesn't require finding it scattered in handler code.

Analytics events

Instrument every state transition

demo_drop, demo_render_success, demo_render_fail (with reason), demo_save_modal_shown, demo_save_modal_dismissed, demo_signup_completed, demo_reprompt_shown, demo_reprompt_signup. PostHog funnel = conversion truth.
