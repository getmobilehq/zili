Zili · Journey Map
v0.1 · May 2026

Journey.

End-to-end across every stakeholder. Where the product lives, where the loops live, where the gaps live.

— Purpose

This map shows the path each user category takes through the product — from first encounter to advocacy or churn. It surfaces the moments that matter (the magic ones and the painful ones), the conversion points between categories, and the features the existing PRD has under-specified. It is the input to engineering scoping, not the output of it.

00 — Stakeholders

Four categories. Four journeys.

Every Zili user falls into one of four categories. Volume scales inversely with depth of relationship: many viewers, fewer customers, one operator.

01

Anonymous visitor

Lands on zili.xyz without prior context. Marketing site is the entire surface. Goal: convert to signup.

05 — Insights

What the map reveals.

Cross-stakeholder patterns. Growth loops worth investing in. Friction points worth eliminating. New work surfaced that wasn't in the PRD or Gherkin.

Growth loops

LOOP 01 — Primary

The watermark loop

Customer creates share → Anonymous viewer sees the deck → Watermark and end-of-deck CTA convert viewer → Viewer signs up → Becomes Customer → Loop. This is the single most important loop. Every product decision should be evaluated against whether it strengthens or weakens this loop.

LOOP 02 — Secondary

The demo loop

Anonymous visitor lands on zili.xyz → tries the live demo without signup → magic moment → converts to signup. Cheaper top-of-funnel than waiting for a watermark exposure. Requires building the demo experience (currently unspecified).

LOOP 03 — V1.5

The referral loop

Customer advocates → invites colleague with a referral link → both get 1 month free Pro → colleague converts. Cheap, defensible, high-LTV. Not in V1 but should be designed for.

LOOP 04 — Passive

The SEO loop

Public shared decks are indexed by Google → some surface for relevant queries → traffic to zili.xyz/p/{slug} → conversion at deck-view stage. Requires SSR + good OG cards + structured data. Mostly free if architecture is right.

Critical friction

FRICTION 01

First-upload mode failure

If auto-detection picks the wrong mode on a user's first artifact, the magic moment dies. Recovery UX must be one click, prominent, and reassuring. Currently in Gherkin as a feature but the UX is undefined.

FRICTION 02

Anonymous viewer discovery gap

Viewers see beautiful decks but no obvious "what is this?" affordance. The single highest-volume user category is being under-served at the conversion moment. End-of-deck CTA + referrer-aware landing closes this.

FRICTION 03

Month-4 paywall surprise

If the grandfather window ends without sufficient warning, users churn at a high rate. Banner sequence (T-30d → T-1d) plus one-click upgrade with saved payment is required, not optional.

FRICTION 04

Operator first incident

The first DMCA, security event, or major bug will be high-stress. Without pre-written runbooks and customer-comms templates, the response will be slower and less consistent than it needs to be. Pre-launch work, not post-launch reaction.

The conversion ladder

Each user category converts upward, not downward. The journey map makes the ladder visible. Each rung is a moment we can either invest in or leave to chance.

RUNG 01 →   Anonymous viewer becomes Anonymous visitor   (end-of-deck CTA, watermark, referrer landing)
RUNG 02 →   Anonymous visitor becomes Customer (Free)   (live demo, simple signup, first-upload magic)
RUNG 03 →   Customer (Free) becomes Customer (Pro)   (grandfather countdown, watermark removal, custom slug)
RUNG 04 →   Customer becomes Advocate   (referral program, share affordances, V1.5)

What this unblocks

With the journey map locked, engineering scoping has full context. The MVP sprint plan can now be built knowing: which features serve which stakeholders, where the magic moments live, where the growth loops are, what new work the PRD missed, and how operations supports the whole system from day one.
